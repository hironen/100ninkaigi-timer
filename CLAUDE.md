# 100ninkaigi

100人会議プロジェクト。

## 概要

地域コミュニティ「100人カイギ」のイベント登壇で使う **10分タイマーアプリ**。

### 背景
- 100人カイギは、地域で活動するゲストに「生き様」を語ってもらう地域コミュニティイベント。
- 1回のイベントにつき 5 名が登壇し、合計 100 名（= 5 名 × 20 回）に達したら開催を終了する。
- 登壇は 1 人あたり 10 分。

### このアプリの役割
- 登壇 1 人分の持ち時間「10 分」を計測するタイマー。
- 登壇者・運営が残り時間をひと目で把握できるようにすることが目的。

## アーキテクチャ方針

- **1枚の HTML ファイル**に HTML / CSS / JavaScript をすべて同梱する（インライン `<style>` / `<script>`）。
- ビルド工程・依存パッケージ・ネット接続なしで、**ファイルをブラウザで開くだけ**で誰でもどこでも動くことを最優先とする。
- 外部 CDN・npm・フレームワークに依存しない（バニラ JS / CSS）。フォントや画像も極力インライン化（data URI 等）するか同梱不要にする。

## リポジトリ構成

- **本リポジトリに、製品コードと計画/仕様ドキュメントを集約**している（旧 `100ninkaigi`（計画）と `100ninkaigi-timer`（実装）を 1 つに統合）。
- タイマー本体は同階層の **`timer.html`**（単一 HTML）。PWA 関連は `manifest.webmanifest` / `sw.js` / `icons/`、アイコン生成は `tools/gen-icons.cjs`。
- 製品仕様（as-built / 実装どおり）は `spec.md` を正とする。タスク管理は `Plans.md` を SSOT とする。

## 開発方針

- タスク管理は `Plans.md` を SSOT（Single Source of Truth）とする。
- 計画作成は `harness-plan`、実装は `harness-work`、進捗同期は `harness-sync` を使う。

## ビルド・テスト

- ビルド不要。`timer.html` をブラウザで開いて動作確認する（`?selftest` 付きで開くと純粋ロジックの境界値が console 検証される）。
- 動作確認の最低ライン: 主要ブラウザ（Chrome / Safari / Firefox）でタイマーが 10 分を計測できること。
- 計時は `performance.now()` アンカー方式（`setInterval` は表示更新のみ・`visibilitychange` 補正あり）でバックグラウンドでもずれない。
- ※ TypeScript 化（esbuild→単一 `timer.html` 生成）は `Plans.md` Phase 3 で計画中。導入後は「ソースはビルド／配布物は開くだけで動くを維持」へ本方針を更新する。

## コーディング規約

- バニラ JavaScript（ES2015+）。トランスパイル前提の構文は使わない。
- 1ファイル構成のため、CSS は `<style>`、JS は `<script>` に記述。可読性のためセクションコメントで区切る。
