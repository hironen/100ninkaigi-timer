## 1. 目的・背景

地域コミュニティイベント「100人カイギ」の登壇で使う **10分タイマー**。
1 イベントで 5 名が順に登壇し、各自の持ち時間 10 分を計測する。
登壇者・運営が残り時間をひと目で把握できることが目的。

## 2. 実装ファイル・配布前提

- 実装は本リポジトリ直下の単一ファイル **`timer.html`**。HTML / CSS / JS をインライン同梱。
- バニラ JavaScript（ES2015+）／バニラ CSS。ビルド・依存パッケージ・ネット接続・外部 CDN/npm/フレームワークなし。**ファイルをブラウザで開くだけで動く**。
- `viewport-fit=cover` ＋ safe-area-inset 対応で iOS の表示領域に配慮。
- 対象ブラウザ: Chrome / Safari / Firefox。

## 3. 持ち時間・状態

- 持ち時間は定数 `TOTAL_SECONDS = 10 * 60`（= 600 秒）。
- 状態変数: `remaining`（残り秒）、`running`（計測中フラグ）、`timerId`。明示的なステートマシンは持たず、`running` の真偽と `remaining` で表現する。
- 計時方式: **`performance.now()` 実時刻アンカー方式**（2026-06-21〜）。開始時に `endAt = performance.now() + remainingMs` を固定し、残りは毎回 `endAt - performance.now()` から再計算する。`setInterval` は 200ms 間隔の**表示更新のみ**で、計測精度は発火回数に依存しない。
  - `visibilitychange` でタブ復帰時に即補正。バックグラウンドで間引かれても 0:00 を取りこぼさず、復帰時点で実経過に一致する。
  - 一時停止/再開は `remainingMs` を精密保持して再アンカーするため、停止時間分だけ正しく残る。

## 4. 操作

- ボタン 3 つ: **START**（開始/再開）, **PAUSE**（停止＝一時停止）, **RESET**（リセット）。
  - START: 既に running なら無視。setInterval 開始。
  - PAUSE: `stop()` を呼び running=false・interval クリア。`remaining` は保持（＝一時停止）。
  - RESET: 停止し `remaining=600` に戻し、配色クラス・爆発フラグを初期化。
- **画面タップ/クリックでも Start/Pause をトグル**（ボタン以外の領域）。ヒント文「（画面タップでも Start / Pause 切替）」を表示。

## 5. 残り時間の表示

- 残り `> 50` 秒: `m:ss` 形式（例 `10:00`, `9:59`, `1:00`）。秒は 2 桁ゼロ埋め。
- 残り `<= 50` 秒: **秒数のみ**を表示（例 `50`, `49`, … `0`）。
- フォントは幅・高さの両方で頭打ち（`#time: min(33vw, 56vh, 500px)` ＋ `white-space:nowrap`、`#msg: min(12vw, 14vh, 120px)`）し、PC・タブレットいずれでも画面内に収まる（2026-06-20 のレスポンシブ修正）。

## 6. 配色（背景・文字色を body クラスで切替）

| 残り時間 | クラス | 配色 |
|----------|--------|------|
| `> 120s` | （なし） | 基本（黒背景 `#111` / 白文字） |
| `<= 120s` かつ `> 60s` | `.info` | 青背景 `#007bff` / 白文字 |
| `<= 60s` かつ `> 30s` | `.warning` | 黄背景 `#ffd700` / 黒文字 |
| `<= 30s` かつ `> 10s` | `.danger` | 赤背景 `#d40000` / 白文字 |
| `<= 10s` かつ `> 0` | `.blink` | 赤背景の点滅（0.6s アニメーション） |
| `0` | （クラスなしに戻る） | 基本色に戻り、爆発演出を発火 |

## 7. メッセージ表示（`#msg`）

`TIME_MESSAGES` を残り時間に対して先頭一致で選ぶ。実効しきい値は以下。

| 残り時間 | メッセージ |
|----------|-----------|
| `> 120s` | （なし） |
| `<= 120s` | そろそろまとめて |
| `<= 60s` | ラストスパート |
| `<= 10s` | まもなく終了 |
| `0` | おつかれさまでした |

## 8. 0:00 到達時の挙動

- `remaining` が 0 以下になったら 0 にクランプして `stop()`（**負値にしない・0:00 で停止**）。
- **爆発パーティクル演出**: 画面中央から 100 個のパーティクル（金 `#ffd700` / 赤橙 `#ff4500` / 白 / 青 `#007bff`）を放射状に飛ばしてフェードアウト（`requestAnimationFrame`）。`exploded` フラグで多重発火を防止、RESET で解除。
- **終了音**: 最後の 5 秒で毎秒「プ」、0:00 で余韻ある「ポーン」（後述 §10）。

## 9. レイアウト・視認性

- `.wrap` は `grid-template-rows: 1fr auto`（上: 時間/メッセージ、下: ボタン）。`min-height:0` ＋ `overflow:hidden` でボタンが画面外に押し出されないようにする。
- 残り時間の数字を大きく中央表示。配色に加え数字・メッセージ文言でも状態が分かる。

## 10. 終了前警告音（WebAudio、2026-06-21〜）

- **解錠**: 「START」または画面タップ（ユーザー操作）内で `AudioContext`（`webkitAudioContext` フォールバック）を生成し `resume()`。autoplay ポリシーで無音にならないようにする。非対応・音不可環境では何もせず視覚（配色・点滅・爆発・文言）のみで継続。
- **発音**:
  - 残り `60s`: 880Hz を 2 回（ピピッ、注意喚起）。
  - **最後の 5 秒**: 残り `5/4/3/2/1` 秒で 1 秒ごとに 880Hz の短い「プ」（square）。
  - `0:00`: 1320Hz の余韻ある「ポーン」（sine・約 0.8 秒のベル風減衰）。
  - 全体で「プ・プ・プ・プ・プ → ポーン」のカウントダウン。`GainNode` をランプしてクリックノイズを防止。
- **多重防止**: `beeped60` / `beeped0` の一度きりフラグと、毎秒1回に絞る `lastCountdownSec`。RESET で再武装。

## 11. 設計・検証（2026-06-21〜）

- **レイヤー分離**: `<script>` を「純粋ドメイン中核（`createTimerCore` ＋ `formatTime`/`colorClass`/`messageFor`/`createSignaller`）」「アダプタ（`Sound`/`Screen`/`Renderer`）」「`App` 配線」に分離。中核は DOM・音・タイマーを知らず単体テスト可能。
- **セルフテスト**: `?selftest` 付きで開くと純粋中核の境界値を `console` 検証（現状 26 項目）。Node の DOM スタブ上でも全 PASS 確認済み。
- DOM・WebAudio・Wake Lock 部分は引き続き手動／実機検証。

## 12. 配布・PWA 方針（計画中・2026-06-21〜）

iPhone / Android で「アプリのように」使うため、既存 `timer.html` を作り直さず **PWA 化**する。ネイティブ App（App Store / Play Store）や React Native 等の作り直しは採らない。

- **オフラインの2モデル**:
  - モデルA（ファイル配置）: `timer.html` は外部依存ゼロのため、端末に置いて開けば**すでに完全オフラインで動く**。
  - モデルB（PWA インストール）: HTTPS で配信し、各端末で「ホーム画面に追加」。Service Worker が app shell をキャッシュ → **以後はネット環境ゼロの会場でもオフライン起動**。アイコン・全画面のアプリ体験を得る。
- **制約**: Service Worker は `file://` では動かず **HTTPS 配信が必須**。よって「**一度だけネットありでインストール → 以後オフライン運用**」が前提（インストール時のみネット要）。
- **iOS PWA 留意点**: Safari は Service Worker 対応（オフライン可）。`apple-mobile-web-app-capable` 等で全画面化。WebAudio は START 操作で解錠済みのため標準動作。10 分間の画面スリープ対策に Screen Wake Lock を対応端末のみ適用。
- 詳細タスクは `Plans.md` Phase 2 を参照。

## 13. スコープ外（やらない）

- App Store / Google Play でのネイティブ配信、Capacitor/Cordova ラップ、React Native/Flutter での作り直し（PWA を採用するため）。
- Next.js 等のメタフレームワーク採用（1画面オフラインタイマーに不要）。
- 登壇者名・人数（100名/20回）カウント・自動連続再生などの登壇者管理。
- サーバ連携・クラウド保存（保存は端末ローカル＝localStorage のみ）。
- キーボードショートカット、多言語対応、プッシュ通知。

## 14. 開発体制・ビルド方針（計画中・2026-06-21〜）

TypeScript 化するが、**配布物 `timer.html` は従来どおりゼロ依存・単一ファイル・開くだけで動く**を維持する。

- **SSOT は `src/`（TypeScript）**。`timer.html` は **esbuild でバンドルし HTML テンプレートに注入して生成する成果物**だが、利用者 DL・PWA 配信が即動くよう **git 追跡してコミット**する。
- esbuild は JS バンドルのみ。Node の薄いビルドスクリプトが `timer.template.html` のプレースホルダへ JS/CSS を**インライン注入**して `timer.html` を出力。`sourcemap:false`、生成物に `import`/`require`/`//# sourceMappingURL`/外部参照を**残さない**（grep で機械検証）。
- devDependency は最小・版固定（`typescript` / `esbuild` ＋ lint/format に `Biome`）。lockfile をコミット。生成物にランタイム依存を持ち込まない。
- 既存の「ビルドなしで開くだけ」という CLAUDE.md 方針は「**ソースはビルドする／配布物は開くだけで動くを維持**」へ更新する。
- ポートは interface 化（`SoundPort`/`ScreenPort`/`RendererPort`/`SettingsRepository`）。`core` は設定非依存（`durationMs`・しきい値・メッセージを注入）。

## 15. 設定モデル（Config / Settings・2026-06-21〜）

色・持ち時間・しきい値・音・メッセージを設定化する。**既定値は `config.ts`（型付き不変 `Config`）**、ユーザー上書きは `Settings` として localStorage に保存。

- **実効設定 = `{ ...config.ts 既定, ...検証済みの localStorage 上書き }`**。初回（未保存）の挙動は**現行 `timer.html` と完全一致**（既定値＝現行ハードコード値）。
- **検証・堅牢性（必須）**:
  - 色は `#rgb` / `#rrggbb` のホワイトリストのみ採用、`style.setProperty` で個別適用（生 CSS 文字列に連結しない＝CSS インジェクション防止）。
  - 数値はクランプ: 持ち時間 60–3600s、音量 gain 0.0001–0.5、周波数 100–4000Hz、秒は整数。`Number.isFinite` チェック。
  - しきい値は単調性 `0 < blink < danger < warning < info ≤ 持ち時間` を強制（崩れたら既定へ）。
  - 不正フィールドは**そのフィールドのみ既定にフォールバック**（全体破棄しない）。`SCHEMA_VERSION` ＋ migrate。壊れた JSON / 旧版 / setItem 失敗（プライベートモード等）でも**例外を投げずタイマーを継続**し、既定化時はユーザーに通知。
- メッセージ等の設定由来テキストは **`textContent` 限定**で出力（`innerHTML` 系は全面禁止＝蓄積型 XSS 根絶）。長さ上限・制御文字除去。

## 16. 管理画面（設定パネル・2026-06-21〜）

`timer.html` 内の**モーダル設定パネル**（別ページにしない＝単一ファイル維持）。

- 編集項目: 配色（`<input type="color">`）／持ち時間／しきい値／音（周波数・長さ・音量）／メッセージ。
- **保存 / キャンセル / 既定に戻す** を備える。キャンセルは確定値を変えない。「既定に戻す」は `config.ts` 既定へ戻し保存もクリア。
- 反映方針: 色は即プレビュー。**持ち時間は保存時／RESET 時に反映**（計測中は変えない）。しきい値の単調性が崩れる入力は**保存不可＋エラー表示**。
- 不正入力でもタイマー動作は継続（ブリックしない）。実機（iOS Safari / Android Chrome）で操作可能。
